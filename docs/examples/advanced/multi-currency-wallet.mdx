This example shows how to model multi-currency user wallets in Formance — giving each user balances in several assets and routing currency conversions through a platform-owned exchange liquidity pool. The schema focuses on the **account structure**: where each asset's balance lives per user, and how the platform's liquidity and FX fees are isolated.

**Common use cases:**
- **Neobanks and fintech wallets** offering users balances in multiple currencies
- **Crypto and FX apps** converting between assets against a house liquidity pool
- **Cross-border platforms** holding local-currency balances per user

<DocCallout type="info">
This is an illustrative example focused on the chart of accounts. Adapt the schema to your specific business requirements, regulatory obligations, and financial practices.
</DocCallout>

## Key Concepts

- **Per-Asset Wallets** — Each user, keyed by `$user_id`, holds a separate `wallet` sub-account per asset (`USD`, `EUR`, `BTC`). Modeling one account per asset keeps balances unambiguous and lets a single user hold many currencies without commingling them.
- **Exchange Liquidity Pool** — The platform funds conversions from its own `exchange:liquidity` accounts, one per asset. A user converting EUR to USD draws from the platform's USD liquidity and returns EUR to it, so the pool always reflects the platform's net FX position.
- **FX Fees** — Conversion spread and fees are captured in a dedicated `exchange:fees` account, keeping FX revenue separate from the liquidity it is earned on.

## Chart of Accounts

The `chart` section defines two account groups: users and the platform exchange.

### Users

<LedgerSchema schema="multi-currency-wallet" chart="users" />

**Normal credit accounts** — each user's `wallet` carries one sub-account per asset (`USD`, `EUR`, `BTC`), keyed by `$user_id`. Each balance is stored value the platform owes that user in that currency.

### Exchange

<LedgerSchema schema="multi-currency-wallet" chart="exchange" />

The platform's FX book. `liquidity:USD` and `liquidity:EUR` hold the house liquidity that funds user conversions; `fees` captures the spread and conversion fees earned. The net of the liquidity accounts is the platform's open FX position.
